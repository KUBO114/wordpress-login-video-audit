セキュリティ監査のためログイン時に短い動画を記録します。動画は安全に保存され、管理者のみがアクセスできます。

== 説明 ==

ログイン動画監査は、ログインプロセス中に1〜2秒の短い動画をキャプチャするセキュリティプラグインです。これにより、管理者はログインアクティビティを監視し、潜在的なセキュリティ脅威を特定できます。

**主な機能：**

* ログイン時に1〜2秒の動画をキャプチャ
* 保護されたディレクトリに安全に保存
* 管理者のみが録画動画を閲覧可能
* 日付による自動ファイル整理
* ユーザー通知によるプライバシー重視
* 軽量で邪魔にならない

**プライバシーとセキュリティ：**

* 動画は.htaccess制限付きの保護されたディレクトリに保存
* `manage_options`権限を持つユーザーのみが動画にアクセス可能
* カメラアクセス要求前にユーザーに通知
* 動画キャプチャ失敗でもログインは妨げられない
* 外部サービスやサードパーティアクセスなし

**使用例：**

* 企業セキュリティコンプライアンス
* 不審なログイン試行の監視
* 機密システムの監査証跡
* 本人確認
* セキュリティインシデント調査

== インストール ==

1. プラグインファイルを`/wp-content/plugins/login-video-audit`ディレクトリにアップロードするか、WordPressプラグイン画面から直接インストールします。
2. WordPressの「プラグイン」画面からプラグインを有効化します。
3. プラグインは自動的に動作します - 設定は不要です。
4. WordPress管理メニューの「ログイン動画」で録画された動画を表示します。

== よくある質問 ==

= このプラグインはすべてのブラウザで動作しますか？ =

MediaRecorder APIをサポートするブラウザ（Chrome、Firefox、Edge、Safari 14.1+）が必要です。ブラウザがサポートしていない場合、動画キャプチャなしでログインが通常通り進行します。

= ユーザーがカメラアクセスを拒否した場合はどうなりますか？ =

ユーザーがカメラアクセスを拒否した場合、または何らかの理由で動画キャプチャが失敗した場合でも、ログインプロセスは通常通り続行されます。動画キャプチャは非ブロッキングです。

= 動画はどこに保存されますか？ =

動画は年月別に整理された`/wp-content/uploads/login-videos/`に保存されます。ディレクトリは.htaccessで保護され、直接アクセスを防ぎます。

= 一般ユーザーは動画を見ることができますか？ =

いいえ。`manage_options`権限を持つ管理者のみが、WordPress管理パネルを通じて録画された動画を閲覧できます。

= 動画ファイルのサイズはどのくらいですか？ =

動画は通常、カメラの品質と圧縮により、1〜2秒のキャプチャで100〜500KBです。最大ファイルサイズ制限は2MBです。

= これはプライバシー規制に準拠していますか？ =

プラグインはカメラアクセス前にユーザー通知を含みます。ただし、地域のプライバシー法（GDPR、CCPAなど）への準拠を確保し、適切なプライバシーポリシー開示を追加するために、法律顧問に相談する必要があります。

= 録画時間をカスタマイズできますか？ =

はい、プラグインファイル内の`LVA_SEC`定数を変更して録画時間を調整できます。

== プライバシーポリシー ==

このプラグインはログインプロセス中に動画をキャプチャし、サーバーに保存します。動画は：

* WordPressインストール内にローカルで保存
* 外部サービスに送信されない
* サイト管理者のみがアクセス可能
* 簡単な管理のために日付別に整理
* 直接のWebアクセスから保護

サイト管理者は以下を行う必要があります：

* プライバシーポリシーに適切な開示を追加
* ユーザーに動画キャプチャについて通知
* 地域のプライバシー規制に準拠
* 適切なデータ保持ポリシーを実装
* 管理者アカウントを適切に保護

== 技術詳細 ==

**ブラウザ要件：**
* MediaRecorder APIサポート
* WebRTCカメラアクセス

**サーバー要件：**
* PHP 7.4以上
* WordPress 5.0以上
* uploadsディレクトリへの書き込み権限

**ファイル形式：**
* VP8コーデック付きWebM動画形式
* 最大ファイルサイズ：2MB
* 通常の長さ：1〜2秒
